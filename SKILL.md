# AI 一人公司執行框架 (SOP)

本技能旨在將「開發」從競爭壁壘中剔除，將重心轉移至 **【洞察 $\rightarrow$ 驗證 $\rightarrow$ 分發】**。

## 🚀 核心工作流

### 第一階段：需求定義 (Definition)
1. **洞察需求**：
    - **目標**：找到一個「真實存在且願意付費」的痛點。
    - **AI 操作**：利用 `web_search` 分析社交媒體 (X, Reddit, 小紅書) 的抱怨點，尋找「目前的解決方案太貴/太慢/太複雜」的場景。
2. **打磨需求**：
    - **目標**：將模糊的痛點轉化為具體的產品功能。
    - **AI 操作**：與 AI 進行角色扮演（如：扮演苛刻的目標用戶），通過對話不斷挑戰需求，剔除偽需求。
3. **總結需求文檔 (PRD)**：
    - **目標**：產出結構化的開發指令。
    - **AI 操作**：生成包含【目標用戶、核心場景、必須有功能 (Must-have)、可以沒有功能 (Nice-to-have)】的 Markdown 文檔。

### 第二階段：極速構建 (Build)
4. **產品原型圖**：
    - **目標**：視覺化產品流程，確認交互邏輯。
    - **工具推薦**：v0.dev (UI 生成), Excalidraw (流程圖), Figma AI。
5. **功能實現**：
    - **目標**：將文檔轉化為可運行的代碼。
    - **工具推薦**：Cursor / Claude 3.5 / Replit。採取「模塊化開發」，一次只完成一個核心功能。
6. **做出 MVP (最小可行性產品)**：
    - **目標**：用最少的功能實現核心價值。
    - **原則**：**砍掉所有非核心功能**。只要能解決那個「痛點」即可發布。

### 第三階段：市場驗證 (Validate)
7. **驗證 MVP**：
    - **目標**：獲取真實用戶的反饋。
    - **操作**：邀請 5-10 個核心用戶內測，記錄他們在不需要引導的情況下如何使用產品。
8. **找到 1000 個用戶**：
    - **目標**：驗證產品是否具有「分發潛力」。
    - **策略**：內容行銷 $\rightarrow$ 社群分發 $\rightarrow$ 產品目錄 (Product Hunt/V2EX) $\rightarrow$ 冷啟動私信。
9. **設計變現點**：
    - **目標**：確認用戶是否願意為此價值付費。
    - **操作**：設計簡單的定價頁面 (Landing Page)，即使功能未完全完成，測試「點擊購買」的轉化率。
10. **驗證閉環**：
    - **失敗 $\rightarrow$ 回到步驟 2 (打磨需求)**：分析是產品不好用，還是需求本身不存在。
    - **成功 $\rightarrow$ 進入規模化**：指標為 $\text{留存率} > X\%$ 且 $\text{獲客成本} < \text{生命週期價值}$。

### 第四階段：規模化 (Scale)
11. **投放市場**：
    - **操作**：增加流量輸入（付費廣告、SEO 矩陣、合作推廣），優化轉化漏斗，建立自動化運營體系。

## ⚠️ 關鍵陷阱 (Pitfalls)
- **過度開發**：在沒有 100 個真實用戶前，禁止花時間優化 UI 或開發「未來功能」。
- **依賴 AI 幻覺**：AI 生成的代碼必須經過實際運行驗證，不能僅看邏輯。
- **無視分發**：記住，**產品 $\approx$ 流量 $\times$ 價值**。如果沒有分發渠道，產品再完美也是 0。
