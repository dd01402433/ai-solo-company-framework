## 💡 核心理念：開發不再是壁壘

在 AI 時代，**「做出產品」已經不再是競爭優勢**。當 v0.dev, Cursor, Claude 3.5 等工具普及後，從想法到 MVP 的開發成本已被壓縮至趨近於零。

**真正的競爭壁壘已發生轉移：**
`傳統創業` $\rightarrow$ 資金 + 團隊 + 長期開發 $\rightarrow$ 產品上線 $\rightarrow$ 驗證
`AI 一人公司` $\rightarrow$ **精準洞察 $\times$ 極速構建 $\times$ 高效分發** $\rightarrow$ 快速迭代 $\rightarrow$ 規模化

## 🛠️ 執行路徑 (SOP)

本框架將創業流程拆解為四個關鍵階段，旨在以最低成本完成從 0 到 1 的驗證。

### 1️⃣ 需求定義 (Definition) —— 尋找真實痛點
| 步驟 | 行動 | AI 賦能方式 | 交付物 |
| :--- | :--- | :--- | :--- |
| **洞察需求** | 挖掘高頻抱怨點 | 分析社交媒體 (X/Reddit/小紅書) 數據 | 痛點清單 |
| **打磨需求** | 剔除偽需求 | AI 角色扮演目標用戶進行壓力測試 | 定義過的痛點 |
| **總結文檔** | 結構化需求 | 自動生成 PRD (產品需求文檔) | `PRD.md` |

### 2️⃣ 極速構建 (Build) —— 消除開發成本
| 步驟 | 行動 | 推薦工具棧 | 交付物 |
| :--- | :--- | :--- | :--- |
| **原型設計** | 視覺化交互邏輯 | `v0.dev` / `Excalidraw` | 低保真原型 |
| **功能實現** | 模塊化代碼開發 | `Cursor` / `Claude 3.5` / `Replit` | 運行代碼 |
| **做出 MVP** | 砍掉非核心功能 | 僅保留解決核心痛點的單一功能 | **MVP 產品** |

### 3️⃣ 市場驗證 (Validate) —— 建立回饋閉環
- **驗證 MVP**：邀請 10 位真實用戶，觀察其在無引導情況下的使用路徑。
- **獲客測試**：嘗試在 Product Hunt, V2EX, 或特定社群獲取前 1000 個用戶。
- **變現驗證**：設計簡單的 Landing Page，測試「點擊購買」的轉化率，而非等待功能完備。
- **決策路徑**：
    - ❌ **驗證失敗** $\rightarrow$ 回到【需求定義】階段進行 Pivot (轉向)。
    - ✅ **驗證成功** $\rightarrow$ 進入【規模化】階段。

### 4️⃣ 規模化 (Scale) —— 放大價值
- **流量輸入**：啟動 SEO 矩陣、付費廣告或內容營銷。
- **漏斗優化**：優化從 $\text{流量} \rightarrow \text{註冊} \rightarrow \text{付費}$ 的每一個環節。
- **自動化運營**：利用 AI Agent 處理客戶支持與基礎運營。

## ⚠️ 避坑指南 (Pitfalls)

- 🚫 **禁止過度開發**：在沒有 100 個真實用戶前，不要花時間優化 UI 或開發「未來功能」。
- 🚫 **禁止無視分發**：記住，產品 $\approx$ 流量 $\times$ 價值。沒有分發渠道，產品再完美也是 0。
- 🚫 **禁止盲信 AI**：AI 生成的代碼必須經過實際運行驗證，不能僅僅通過邏輯審查。
